HTML e CSS como uma obra

Quando comecei a entender desenvolvimento web, uma comparação deixou tudo mais claro: um site funciona muito como uma obra. Primeiro vem a estrutura. Depois, o acabamento.

A lógica da comparação

Na construção civil, ninguém começa escolhendo a tinta antes de saber onde a parede vai existir. Primeiro vem a organização do espaço, a estrutura e a função de cada elemento. Na web, acontece algo parecido.

O HTML organiza a estrutura da página. O CSS cuida da aparência. Em outras palavras: HTML é a base; CSS é o acabamento.

HTML: a estrutura

O HTML define o que cada elemento é: título, parágrafo, imagem, lista, botão. Ele não está preocupado, em primeiro lugar, com beleza. Ele está preocupado com função e organização. 

<h1>Meu primeiro site</h1>
<p>Estou aprendendo HTML e CSS.</p>
<img src="imagem.jpg" alt="Uma mesa com notebook">

CSS: o acabamento

O CSS entra depois da estrutura. É ele que define cor, fonte, espaçamento, alinhamento e layout. Na analogia da obra, ele funciona como pintura, revestimento e acabamento visual. 

h1 {
  color: #8f5aa8;
  text-align: center;
}

p {
  font-size: 18px;
  color: #444;
}

Fun fact histórico

O CSS foi proposto em 1994 por Håkon Wium Lie, e a primeira recomendação oficial do W3C saiu em 1996. A ideia era separar estrutura e aparência para deixar a web mais organizada, mais limpa e mais fácil de manter.

Entender HTML e CSS dessa forma fez a web parecer menos abstrata para mim. No fim, a lógica é bem familiar: primeiro se constrói a base; depois se resolve o acabamento.
